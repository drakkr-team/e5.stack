## Overview

**e5.stack** is a production-ready monorepo built with **pnpm** and **Turborepo**. It pairs an **AdonisJS** backend API with a **TanStack Start** React frontend, connected via **Tuyau** for fully typed end-to-end data contracts. UI primitives live in shared packages powered by **Tailwind CSS** and **Storybook**.

## Features

- **End-to-End Type Safety:** AdonisJS and TanStack Start are linked via **Tuyau**, giving you typed API queries and mutations on the frontend with zero manual contract maintenance.
- **Domain-Driven Backend:** The API uses a feature-first architecture (e.g., `src/features/user_management`) instead of flat controller/service trees.
- **Modern Frontend:** TanStack Start provides file-based routing, SSR-friendly data fetching, and React 19 support.
- **Shared UI Packages:** Independent packages for Storybook-backed React components and generated Tailwind theme tokens.
- **Internationalization Ready:** Built-in i18n compilation workflow with French (`fr`) locale provided out of the box.
- **Developer Experience:** Fast task orchestration with Turborepo, unified linting and formatting via Biome, and Docker Compose for local database services.

## Getting Started

### Prerequisites

- **Node.js** `v24.x` (enforced via `.npmrc`)
- **pnpm** `v10.33.2`
- **Docker & Docker Compose** (for local database services)

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/your-org/e5.stack.git
   cd e5.stack
   ```

2. **Install dependencies:**
   ```bash
   pnpm install
   ```

3. **Start the development environment:**
   ```bash
   pnpm dev
   ```

> [!NOTE]
> The API relies on a Docker Compose sidecar task. Ensure the Docker daemon is running before executing `pnpm dev`.

## Commands

Run these from the root directory:

| Command | Description |
| :--- | :--- |
| `pnpm dev` | Starts the entire application stack in development mode. |
| `pnpm build` | Compiles and builds all apps and packages. |
| `pnpm typecheck` | Runs TypeScript type checking across the workspace. |
| `pnpm test` | Executes backend test suites. |
| `pnpm code-quality` | Runs Biome linting and formatting checks. |
| `pnpm code-quality:fix` | Auto-fixes linting and formatting issues using Biome. |
| `pnpm adonis` | Forwards commands to the AdonisJS Ace CLI. |

### Targeted Execution

Work on a specific part of the stack using pnpm filters:

```bash
pnpm --filter @workspace/api dev        # Run only the API
pnpm --filter @workspace/web dev        # Run only the Web App
pnpm --filter @workspace/ui-react dev   # Start Storybook for UI development
pnpm --filter @workspace/ui-theme generate:tailwind # Regenerate Tailwind tokens
```

## Architecture

<details>
<summary><strong>Backend (AdonisJS)</strong></summary>

- **Feature-First Layout:** Controllers, routes, and validators are grouped by domain under `src/features/` rather than by technical layer.
- **Security:** JSON-only responses are enforced via `force_json_response.middleware.ts`. Do not send HTML responses from the exception handler.
- **Generated Code:** Do not manually edit generated artifacts such as `ace.js`, files in `.adonisjs/`, or `database/schema.ts`.

</details>

<details>
<summary><strong>Frontend (TanStack Start)</strong></summary>

- **Routing:** TanStack Router uses file-based conventions (`layout.tsx`, `page.tsx`). `routeTree.gen.ts` is auto-generated and should not be edited manually.
- **Data Fetching:** The frontend uses a QueryClient-backed router paired with Tuyau (`src/libs/tuyau.ts`) for end-to-end typed queries and mutations.
- **i18n:** Localization strings live in `src/**/locales/*.json` and are compiled into `src/libs/i18n/build/`. French (`fr`) is the default locale module.

</details>

<details>
<summary><strong>UI & Theming</strong></summary>

- **Separation of Concerns:** `packages/ui/react` contains layout primitives and components. `packages/ui/theme` is the single source of truth for design tokens (`tokens.ts`).
- **Tailwind Workflow:** Edit tokens in `src/tokens.ts`, then regenerate `tailwind.css` using `pnpm --filter @workspace/ui-theme generate:tailwind`. Never hand-edit the generated CSS file.
- **Component Standard:** UI component folders follow a standard structure of `component.tsx`, `index.ts`, and `component.stories.tsx`.

</details>
