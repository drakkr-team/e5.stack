Cette page sert de modele pour la partie produit d'une documentation.
Elle doit rester lisible sans connaitre l'implementation.

## Objectif

Decrire le besoin utilisateur, le contexte metier et le resultat attendu.

## Parcours

- Preciser qui utilise la fonctionnalite.
- Decrire les etapes importantes du parcours.
- Lister les decisions ou contraintes visibles pour l'utilisateur.

## Criteres d'acceptation

- Le comportement attendu est explicite.
- Les cas limites importants sont identifies.
- Les termes metier sont comprehensibles sans lire le code.
